# Week 14 — Macros & program structure

**Goal:** stop repeating yourself. NASM's preprocessor turns boilerplate
into vocabulary, and multi-file linking turns your lib.inc into a real
library.

## Read (~40 min)

### %define: named constants and one-liners

```nasm
%define SYS_WRITE   1
%define SYS_EXIT    60
%define STDOUT      1
%define O_RDONLY    0
```

Every magic number from weeks 1–13 deserves a name. (`equ` also does
constants — the practical split: `%define` is textual and can take
parameters; `equ` defines a symbol tied to assembly-time values like
`$ - msg`. For plain numbers either works; pick one style and stay with it.)

### %macro: instruction sequences with parameters

```nasm
; syscall wrappers — the boilerplate you've typed 100 times by now
%macro WRITE 3              ; 3 = parameter count; %1 %2 %3 = the args
        mov rax, SYS_WRITE
        mov rdi, %1
        lea rsi, [%2]
        mov rdx, %3
        syscall
%endmacro

%macro EXIT 1
        mov rax, SYS_EXIT
        mov rdi, %1
        syscall
%endmacro

_start:
        WRITE STDOUT, msg, msglen
        EXIT 0
```

Suddenly your programs read like scripts. Two sharp edges:

- **Labels inside macros** collide when the macro is used twice — prefix
  them `%%loop` and NASM makes each expansion's label unique.
- Macros are *textual*: `WRITE` above clobbers rax/rdi/rsi/rdx invisibly at
  the use-site. Either document that loudly or save/restore inside. This is
  the eternal macro-vs-function trade-off: inline speed vs hidden effects.

Also nice: `%macro PUSH_MANY 1-*` takes variable arguments (`%0` = count,
loop with `%rep`), and `%rep 8 / db 0 / %endrep` unrolls at assembly time.

### From %include to real linking

`%include "lib.inc"` pastes text — every binary re-assembles the library.
The grown-up way: assemble separately, link together.

```sh
nasm -f elf64 -g -F dwarf lib.asm -o lib.o     # library: no _start!
nasm -f elf64 -g -F dwarf w14.asm -o w14.o
ld -o w14 w14.o lib.o
```

Two keywords make symbols cross file boundaries:

```nasm
; in lib.asm             ; in w14.asm
global print_uint        extern print_uint
global print_str         extern print_str
```

`global` exports, `extern` imports — you met both in week 13; now you're
using them between your *own* files. Undefined-symbol link errors and their
cousins ("multiple definition") become old friends this week.

## Type (~60 min)

Restructure for the capstone (this is real refactoring — go slow):

1. Create `syscalls.inc`: `%define`s for every syscall number and fd you've
   used, plus the WRITE/EXIT macros (with a comment: "clobbers rax rdi rsi
   rdx").
2. Convert `lib.inc` → **`lib.asm`**: add `global` for every function,
   assemble it standalone. It has no `_start` — it's pure library. (Keep
   lib.inc around or delete it; your call as maintainer.)
3. `make new n=w14`: a demo program using `extern`, the macros, and at
   least three lib functions. Link it by hand with the two-step commands.
4. Teach your **Makefile** the pattern — add:
   ```makefile
   %: %.asm lib.o
   	nasm -f elf64 -g -F dwarf $< -o $*.o
   	ld -o $@ $*.o lib.o

   lib.o: lib.asm
   	nasm -f elf64 -g -F dwarf lib.asm -o lib.o
   ```
   (Place this rule *above* the old `%: %.asm` one and keep both — make
   prefers the first matching rule; older single-file weeks still build.
   If anything fights you, `make -d` shows make's reasoning — or just ask
   your resident AI to help wire it in.)

5. Prove the whole chain: `make run f=w14`, then `make debug f=w14` and
   `b print_uint` — breakpoints in a separately-assembled file work because
   every .o carried its own DWARF.

## Solve (~20 min)

1. **%%label drill**: write a `PRINT_N_TIMES str, n` macro containing a
   loop; use it twice in one program. First with a normal `.loop` label
   (enjoy the error), then with `%%loop`.
2. **DEBUG_REG macro**: `DEBUG_REG rax` prints `rax = <value>` using
   print_str + print_uint, and — the hard part — leaves *every* register
   exactly as it found them (push/pop everything you touch, remembering
   what print_* clobber). This macro is a gift to all your future weeks.
3. **Assembly-time tables**: with `%rep` + a counter, build a `dq` table of
   the first 16 squares at *assembly* time (`%assign` is your loop
   variable). `x/16gx &table` in gdb to admire — zero runtime cost.

## Tool tip of the week

See what the preprocessor actually did: `nasm -E w14.asm | less` dumps the
post-expansion source — every macro unrolled, every %define substituted.
When a macro misbehaves, this is the x-ray. (`nasm -l w14.lst` gives a
listing with addresses and generated bytes next to your source — pure gold.)
